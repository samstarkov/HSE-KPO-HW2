# Домашнее задание 2 по курсу "Конструирование ПО"

## О проекте
Консольное приложение для управления личными финансами, разработанное с применением принципов SOLID, GRASP и паттернов проектирования GoF. Выполнено на языке C# (версия .net - 8.0). Основные возможности:

- **Управление банковскими счетами**: создание, просмотр списка, удаление счетов
- **Работа с категориями операций**: создание категорий доходов и расходов, просмотр списка
- **Учет финансовых операций**: регистрация приходных и расходных транзакций с автоматическим пересчетом баланса счетов
- **Финансовая аналитика**: расчет чистого дохода за период и анализ операций по категориям
- **Импорт данных**: загрузка информации из файлов CSV, JSON и YAML форматов
- **Экспорт данных**: выгрузка данных в форматы CSV, JSON и YAML
- **Интуитивный интерфейс**: консольное меню с валидацией вводимых данных

## Принципы проектирования

### SOLID принципы

- **Принцип единственной ответственности (SRP)**  
  Каждый класс системы выполняет строго одну задачу

- **Принцип открытости/закрытости (OCP)**  
  Система расширяема для новых форматов импорта/экспорта без модификации существующего кода

- **Принцип подстановки Барбары Лисков (LSP)**  
  Интерфейсы `IRepository<T>` и `ICommand` позволяют свободно заменять реализации

- **Принцип разделения интерфейсов (ISP)**  
  Все интерфейсы содержат минимально необходимый набор методов

- **Принцип инверсии зависимостей (DIP)**  
  Высокоуровневые компоненты зависят от абстракций, DI-контейнер управляет зависимостями

### GRASP принципы

- **Высокая связность и слабое зацепление**  
  Логически связанный функционал сгруппирован, взаимодействие через интерфейсы

- **Контроллер**  
  Класс `Menu` координирует взаимодействие между пользователем и системой

## Паттерны проектирования GoF

- **Фабрика (Factory)** - `Factories/DomainFactory`  
  Централизованное создание объектов предметной области с проверкой бизнес-правил

- **Заместитель (Proxy)** - `Repositories/ProxyRepository<T>`  
  Кэширующий слой поверх основного хранилища для повышения производительности

- **Фасад (Facade)** - `Facades/*Facade`  
  Упрощенный интерфейс для работы со сложными подсистемами

- **Команда (Command)** - `Commands/*Command`  
  Инкапсуляция пользовательских действий в объекты команд

- **Шаблонный метод (Template Method)** - `ImportExport/DataImporter`  
  Единый алгоритм импорта с вариативными этапами парсинга

- **Посетитель (Visitor)** - `ImportExport/Export`  
  Унифицированный механизм экспорта объектов в различные форматы
